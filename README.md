## What Is AtollSphere?

AtollSphere is a geopolitical signals intelligence (GSINT) brief platform built on a structured analytical framework. It produces edition-based PDF intelligence briefs covering active conflict scenarios, using a disciplined sourcing hierarchy, analytical heuristics, and a prediction tracking system with post-mortem accountability.

The platform is designed to run as a daily or twice-daily sweep cycle, producing publication-ready PDF outputs via a ReportLab build pipeline.

## How To Run a Session

### Starting a new chat session

1. Open a new Claude Sonnet chat
2. Paste the full contents of SESSION_STATE.md as your first message
3. Claude will load the current edition state, open predictions, and active rules
4. Request a sweep — Claude will fetch all 18 feeds, run full CDIT analysis, and build the PDF

### Session state file

SESSION_STATE.md is the single source of truth for where the project is at any given moment. It contains:

- Current edition and war day
- Active cases with status tags and confidence levels
- Open predictions only — closed predictions are in PREDICTION_LOG.json
- Last PMM and active system change rules
- Open build tasks
- Instructions for Claude

Update SESSION_STATE.md at the end of every session before committing.

## Core Framework

### The CDIT Framework

Every case follows a mandatory five-part structure:

| Part | Name | Purpose |
|---|---|---|
| 1 | Fact Table | Raw facts with Gate 0 verification on every row |
| 2 | Observed Incongruity | Heuristic analysis — H1 through H6 applied |
| 3 | Hypothesis Set | Minimum two hypotheses with probability ranges |
| 4 | Truth Quality Check | Assumptions, fragile facts, high-impact uncertainties |
| 5 | Disconfirmation | CONFIRMS/CONTRADICTS pairs with Gate 5 revision triggers |

Each case closes with a Forward Flag — a single falsifiable prediction meeting all five C-criteria.

### The Six Heuristics

| H | Name |
|---|---|
| H1 | Incentive Mismatch |
| H2 | Timing Convergence |
| H3 | Beneficiary Asymmetry |
| H4 | Narrative vs Outcome Gap |
| H5 | Structural Contradiction |
| H6 | Suppressed Intersection |

### Source Tiering

| Tier | Category | Feeds |
|---|---|---|
| 1 | Primary Source | 5 named feeds |
| 2 | High-Authority Secondary | 7 named feeds |
| 3 | Watch / Corroboration Required | 4 named feeds |
| 4 | Editorial Assembly | 2 named feeds |

Full source registry: SOURCE_REGISTRY.json

### Prediction Accountability

Every prediction is tracked in PREDICTION_LOG.json with:

- Five-state outcome: CONFIRMED / CONTRADICTED / PARTIAL / AMBIGUOUS / UNRESOLVABLE
- Full status history across editions
- PMM (Post-Mortem Module) triggered on CONTRADICTED or UNRESOLVABLE
- Rule changes generated by PMMs logged permanently in SYSTEM_CHANGE_LOG.md
